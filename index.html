1 Ordem de instalação (WireGuard primeiro, depois WARP)

O cliente oficial da Cloudflare (cloudflare-warp) utiliza sua própria implementação de túnel (BoringTun) e não depende do pacote wireguard-tools do sistema para funcionar. No entanto, para os objetivos deste manual – extrair as chaves WireGuard e depois configurar uma interface nativa (wg0) utilizando as ferramentas wg e wg-quick – é necessário que o WireGuard esteja instalado antes da extração. Além disso, como o WARP será removido após a extração (para evitar conflitos de rota), a ordem sequencial proposta (primeiro WireGuard, depois WARP) é a mais limpa e didática.

✔ Portanto, a ordem WireGuard → WARP é uma recomendação de fluxo, não uma imposição técnica. O WARP funcionaria isoladamente, mas para nossa configuração manual final, precisamos do WireGuard nativo.

2 Instalar WireGuard nativo

sudo apt update
sudo apt install wireguard wireguard-tools -y
sudo modprobe wireguard
✔ Verifique o módulo: lsmod | grep wireguard.

3 Instalar Cloudflare WARP

sudo apt install cloudflare-warp -y
sudo systemctl start warp-svc
warp-cli registration new
warp-cli mode tunnel
warp-cli connect

Teste rápido: curl ifconfig.me → deve exibir um IP da Cloudflare (ex.: 104.16.x.x).

4 Extrair credenciais (wgcf)

curl -L https://github.com/ViRb3/wgcf/releases/download/v2.2.22/wgcf_2.2.22_linux_amd64 -o wgcf
chmod +x wgcf
./wgcf register
./wgcf generate

O arquivo wgcf-profile.conf contém a chave privada, endereços IP, DNS e endpoint. Guarde esses valores.

5 Configurar wg0.conf com proteção semântica

Proteção semântica: usar umask 077 em subshell para criar o arquivo com permissões 600.

sudo mkdir -p /etc/wireguard
cd /etc/wireguard
sudo sh -c 'umask 077 && touch wg0.conf'
sudo nano wg0.conf

Conteúdo exemplo baseado no wgcf-profile.conf (valores modificados e aleatórios – não utilizar diretamente):

[Interface]
PrivateKey = ChavePrivadaModificada_9f8s7d6f5g4h3j2k1l0
Address = 172.31.87.14/32, 2606:4700:1234:5678:90ab:cdef:1234:5678/128
DNS = 1.1.1.1, 2606:4700:4700::1111
MTU = 1280

[Peer]
PublicKey = ChavePublicaModificada_sMI5Q9zLxVbNcXzAqWertyUiop
AllowedIPs = 0.0.0.0/0, ::/0
Endpoint = engage.cloudflareclient.com:2408
🔁 Os endereços 172.31.87.14/32 e 2606:4700:1234:5678:90ab:cdef:1234:5678/128 são exemplos aleatórios semanticamente válidos. No ambiente real, utilize os endereços exatos fornecidos pelo wgcf-profile.conf.

Reforce a permissão:

sudo chmod 600 /etc/wireguard/wg0.conf

6 Remover WARP oficial (evitar conflito)

Como utilizaremos apenas a interface nativa wg0, removemos o cliente oficial para eliminar qualquer conflito de rotas ou serviços concorrentes.

warp-cli disconnect
sudo systemctl disable warp-svc
sudo systemctl stop warp-svc
sudo apt purge cloudflare-warp -y
sudo apt autoremove -y

7 DNS persistente – método original (dns.config + chattr +i + symlink)

Após subir a interface wg0, o WireGuard atualiza /etc/resolv.conf, mas o sistema pode sobrescrevê-lo no reboot. A solução original e semanticamente correta (sem remover arquivos) é:

7.1 Subir a interface wg0 e capturar o DNS gerado

sudo wg-quick up wg0
cat /etc/resolv.conf   # anote os nameservers (ex.: 1.1.1.1, 2606:4700:4700::1111)

7.2 Criar o arquivo /etc/dns.config com proteção semântica

sudo sh -c 'umask 077 && touch /etc/dns.config'
sudo nano /etc/dns.config

Conteúdo exemplo (use os servidores observados no passo anterior):

nameserver 1.1.1.1
nameserver 1.0.0.1
nameserver 2606:4700:4700::1111

7.3 Tornar o arquivo imutável (chattr +i)

sudo chattr +i /etc/dns.config

7.4 Redirecionar o link simbólico /etc/resolv.conf para o arquivo imutável

Sem remover o original – o comando ln -sf substitui o link existente de forma atômica:

sudo ln -sf /etc/dns.config /etc/resolv.conf
✔ Esta abordagem preserva a estrutura padrão do sistema, não utiliza rm e garante que o DNS persista mesmo após reinicializações. Para editar o dns.config no futuro, será necessário primeiro remover o atributo imutável: sudo chattr -i /etc/dns.config.

8 Habilitar WireGuard na inicialização

sudo systemctl enable wg-quick@wg0
sudo systemctl start wg-quick@wg0

9 Validação final

  • IP mascarado: curl ifconfig.me → IP da Cloudflare.
  • DNS persistente: cat /etc/resolv.conf → deve mostrar os servidores configurados no /etc/dns.config.
  • Imutabilidade: lsattr /etc/dns.config → exibe o atributo i.
  • Resolução de nomes: nslookup google.com → o servidor deve ser 1.1.1.1 (ou o definido).
  • Proteção do wg0.conf: ls -l /etc/wireguard/wg0.conf → permissões -rw------- (600).
  • Teste de reboot: Reinicie o servidor e repita os testes – o túnel deve estar ativo e o DNS preservado.
